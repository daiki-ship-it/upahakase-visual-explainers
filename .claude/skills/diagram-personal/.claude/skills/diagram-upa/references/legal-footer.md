# 利用条件・著作権フッター（運用メモ）

## 正本（SSoT）

- **条文・マークアップ**: [legal-footer-snippet.html](legal-footer-snippet.html) のみを編集する。ここにない文言をチャットや SKILL 本文に「代替条文」として書かない（エージェントは正本をコピーする）。
- **完成見本 HTML**（`output/ai-daily-report-slack-apr2026/index.html` など）は、正本にプレースホルダを埋めた結果と**一致**させる。条文の手修正は snippet 経由で行う。

## プレースホルダ

| プレースホルダ | 意味 | 例 |
|----------------|------|-----|
| `__UPA_LEGAL_OFFICIAL_URL__` | その資料の公式配布URL（絶対URL）。`この資料の公式配布ページ` の **2箇所の `href` で同一** | `https://upa-ai-daily-slack-apr2026.surge.sh/` |
| `__UPA_LEGAL_LAST_UPDATED__` | 「最終更新：」**直後**の日付表示のみ | `2026年4月9日` |

デプロイ先が変わったら、当該 `index.html` 内の URL のみ差し替えでよい（条文本文は変えない）。

## 標準フロー

1. 新規図解: 完成見本を複製するとフッター付き。URL・日付だけ資料に合わせて置換。
2. 正本を更新したら: `legal-footer-snippet.html` を編集 → 各成果物 HTML のフッター全体を展開後のブロックで置き換え。

## 漫画風派生版（オプション）

同一内容を漫画風スタイルで出すページでは、**著作権第1段落**（「本資料（文章・構成…」の `<p>`）の**直後**に、次の1段落を**追加**してよい。それ以外のフッターは正本と同一（リンクは派生ページ自身の公式URLを `__UPA_LEGAL_OFFICIAL_URL__` に使う）。

```html
      <p class="m-0">本ページは、同一の著作内容を漫画風の見た目で表示した派生版です。以下の利用条件・著作権は、当該内容の通常版と同一に適用されます。紹介・シェアでは、可能な限り<strong class="text-slate-800">本ページのURL</strong>（または通常版の公式URL）のいずれかを明示してください。</p>
```

通常版（`index.html`）にはこの段落を**入れない**。

## 部分生成・チャンク結合

`main` だけを生成して結合する場合も、**最終 HTML の `</main>` 直後**に必ず正本フッター全体を挿入する。プレースホルダは結合完了前にすべて置換する。

第1章：毎朝の「進捗確認グセ」

パニっくん

博士！ボク、毎朝起きるたびに「あの案件どこまで進んでるっけ」って不安になって、Notionを開いて、タスクを1個1個確認して……それを自分でSlackにまとめて報告するのが習慣になっちゃってるんですよ。しんどい！

ウパ博士

パニさん、その作業……全部AIに任せられますよ。私が使っているai-launch-dashboardというしくみがあれば、毎朝決まった時間にNotionのタスクを自動で集めて、AIに分析させて、ダッシュボードに変換して、スクリーンショットを撮って、Slackに投稿するまでを、人間が何もしなくても全部やってくれます。

パニっくん

えっ！？Notionを見て、まとめて、Slackに送るまで全部!? それって……ロボットが代わりに朝礼やってくれるってこと？

ウパ博士

まさに、そのイメージです。仕組みを順番に見ていきましょうか。

第2章：5ステップのパイプライン全体像

ウパ博士

このしくみは、5つのスクリプト（プログラムの手順書）が順番に走ることで動いています。流れはこうです。

5ステップのパイプライン
  1. ① fetch-notion-data.js — NotionのデータベースからタスクをAPI経由で取得し、JSONファイルに保存
  2. ② generate-ai-suggestions.js — ①のJSONをGemini AIに渡して「優先度の高い提案3つ」を生成し、別のJSONファイルに保存
  3. ③ generate-dashboard.js — ①②のJSONを読み込み、タスク一覧・AI提案・8週間カレンダーを含むHTMLダッシュボードを生成
  4. ④ take-screenshot.js — Playwright（自動ブラウザ）でHTMLを開き、3枚のスクリーンショットを撮影
  5. ⑤ post-to-slack.js — SlackのAPIを叩いて、「@channel」メンション＋3枚の画像を投稿

パニっくん

……えっと、「JSON」とか「Playwright」とかちょっとわからないんですけど。これって全部繋がってるんですか？

ウパ博士

はい、全部繋がっています。①の出力が②の入力になり、②の出力が③の入力になる——まるでベルトコンベアのようにデータが流れていく構造です。各ステップを順番に見ていきましょう。

第3章：各ステップの中身

① Notionからデータを取得（fetch-notion-data.js）

ウパ博士

最初のステップは、Notionのデータベースにアクセスしてタスク情報を引き出してくることです。

Notionには「API」というお問い合わせ窓口があって、「このデータベースのタスクを全部ください」とリクエストすると、JSON形式——いわばプログラムが読めるメモ帳——でタスク一覧を返してくれます。

パニっくん

Notionって、ウェブサイトから見るものじゃないんですか？プログラムからも読めるの？

ウパ博士

そうです。画面から人間が見るのと、APIを通じてプログラムが読むのは、どちらも同じNotionのデータを見ています。違うのは「窓口」だけです。

このステップでは、タスクの名前・締め切り・進捗率・ステータス・メモを取得して、「サブタスク」は除外し、親タスクだけを整理して保存します。締め切りが一番近いタスクも自動で判定しますよ。

パニっくん

つまりNotionのタスク情報が「プログラムが読めるメモ帳」に変換されるわけですね。次のステップでそれを使うんですか？

ウパ博士

正確です。

② Gemini AIが提案を生成（generate-ai-suggestions.js）

ウパ博士

次は、①で取得したタスク一覧をGoogleのGemini AI（Googleが作っているAI）に渡して、「今のプロジェクト状況をPMの目線で分析して、優先度の高い順に3つの提案をください」と日本語で頼みます。

Geminiが返してくれた結果は「高・中・低」の優先度付き提案として、また別のJSONファイルに保存されます。

パニっくん

えっ！タスクの一覧をAIに投げたら、「これを優先してください」って提案まで自動でやってくれるんですか？！

ウパ博士

はい。それと、万が一GeminiのAPIが落ちていても安心なように、「フォールバック」という保険が組み込まれています。危険なタスクの数を数えて、機械的に汎用的な提案を自動生成する仕組みです。AIが使えないときでも、何かしらの提案は必ず出てきます。

③ HTMLダッシュボードを作成（generate-dashboard.js）

ウパ博士

①と②のJSONを両方読み込んで、ウェブページ（HTML）を1枚生成するのが③です。

このHTMLには5つのセクションが含まれています。

ダッシュボードの構成
  1. 緊急アラートカード — 締め切りが一番近いタスクを赤枠で強調表示
  2. タスク一覧 — 「順調 / 注意 / 危険」のサマリーと、各タスクのプログレスバー・ステータス
  3. AI提案カード — Geminiが生成した3つの提案（優先度・要約・アクションリスト）
  4. 8週間カレンダー — 今週の日曜日から8週分の日程と、その週に締め切りがあるタスクのバッジ
  5. 凡例 — ステータス絵文字の意味説明

パニっくん

ちょ、待ってください。Notionのデータを元にウェブページが自動で作られるんですか？誰もHTMLを手で書いてないのに？

ウパ博士

そうです。「どんなHTMLを生成するか」というテンプレートはプログラムの中に書かれていて、タスクの内容だけが毎回入れ替わります。料理に例えると、レシピは固定で、食材だけが毎朝Notionから届くイメージですね。

④ スクリーンショットを撮影（take-screenshot.js）

ウパ博士

③で作ったHTMLをSlackに直接貼ってもきれいに表示されませんよね。なので④ではPlaywright——目に見えない「幽霊ブラウザ」——が自動でHTMLを開いて、3つのエリアに分けてスクリーンショットを撮ります。

具体的には「緊急アラート＋タスク一覧」「AI提案」「カレンダー＋凡例」の3枚です。

パニっくん

マジ？「幽霊ブラウザ」が自動でウェブページを開いてスクリーンショット撮ってくれるって、もはや小人が住んでるじゃないですか！

ウパ博士

……まあ、そういうイメージでも間違ってはないですよ。この「幽霊ブラウザ」はPlaywrightというGoogle系の技術で、Web上の操作を自動化するために作られています。画面はどこにも表示されませんが、裏でしっかり動いていますよ。

⑤ Slackへ投稿（post-to-slack.js）

ウパ博士

最後のステップです。④で撮った3枚の画像を、Slack Web APIを通じて指定したチャンネルに投稿します。「@channel おはようございます！本日のAIローンチ進捗ダッシュボードをお届けします」という一言コメント付きで、3枚の画像がまとめて届きます。

パニっくん

すごい！！Notionのデータが5つの処理を通って、最終的にSlackの画像になるんですね。ベルトコンベアの例えがすごくわかりやすかったです！

第4章：全部つなぐ「自動スケジューラー」

パニっくん

でも博士、①〜⑤のステップを「毎日決まった時間に動かす」には、誰かがボタンを押す必要があるんじゃ……？

ウパ博士

鋭いですね。それを担っているのがGitHub Actions——インターネット上にある「タイマー付きの実行係」です。

コードの保管場所（GitHubリポジトリ）に設定ファイルを1つ置くだけで、「毎日UTC20:00（日本時間の翌朝5時）に①〜⑤を順番に実行してください」というスケジュールを組めます。

パニっくん

パソコンを起動しておく必要もなく、誰かが手動でやる必要もなく、インターネット上のサービスが勝手に動いてくれるってことですか！？

ウパ博士

はい。GitHub Actionsの設定ファイルには、次のことが書かれています。

  1. いつ動くか：毎日UTC20:00（日本時間の翌朝5時）、または手動でも起動可能
  2. 何を使うか：Node.js 20 と Playwright（幽霊ブラウザ）
  3. どのAPIキーを使うか：Notion・Gemini・Slackの鍵（GitHub Secretsで安全に管理）
  4. 何を実行するか：npm run run-all（①〜⑤を一気に走らせるコマンド）

パニっくん

「APIキーって何ですか？鍵って……Notionに鍵があるってこと？

ウパ博士

そうです。プログラムがNotionやSlackにアクセスするときは、「このプログラムは許可された本物ですよ」と証明するための合言葉のような文字列が必要です。これが「APIキー」です。

漏れると第三者に悪用されるので、GitHubのSecrets（金庫）に安全に保管して、実行時だけ取り出す仕組みになっています。

パニっくん

なるほど！合言葉を金庫に入れておいて、必要な時だけ出すんですね。セキュリティもちゃんと考えられてる！

第5章：まとめ — 全体の流れをもう一度

ウパ博士

では全体の流れを整理しましょう。

ai-launch-dashboard の全体フロー
  1. 毎日UTC20:00 — GitHub Actionsのタイマーが発動
  2. ① Notionから取得 — タスク一覧をAPIで収集 → notion-data.jsonに保存
  3. ② Geminiで分析 — タスク状況を渡してAI提案を生成 → ai-suggestions.jsonに保存
  4. ③ HTML生成 — 2つのJSONを合成してダッシュボードHTMLを作成 → dashboard.html
  5. ④ スクリーンショット — Playwright（幽霊ブラウザ）が3枚の画像を撮影 → screenshot-1〜3.png
  6. ⑤ Slack投稿 — 3枚の画像＋コメントを指定チャンネルに自動送信

パニっくん

博士……これって、毎朝ボクがやってたことを全部自動でやってくれるってことじゃないですか。「Notionを開いて確認して、まとめてSlackに報告する」——全部。感動して泣きそうになってきました……。

ウパ博士

ふふ、そうですよ。しくみ自体はシンプルです。「データを取得 → AIが分析 → 見た目を作る → 画像に変換 → 送信」という5つの処理が、ベルトコンベアのようにつながって動いている。それが ai-launch-dashboard の正体です。

大事なのは、それぞれのステップが「前のステップの出力を受け取って、次のステップへ渡す」というシンプルな約束で動いていること。この考え方を押さえれば、似たしくみを自分で作ることもできますよ。

パニっくん

「前の出力が次の入力」！これ覚えます！！自分のタスク管理にも応用できそうな気がしてきた！！
